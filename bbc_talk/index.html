The Right Tool for the Job

Finding the right web technology for your project

By Jonathan Arnett (J3RN)

So you've got an idea

So the hard part is out of the way. Now you need to build it.


For that job, you'll need some talented developers. But what kind of developers should you be looking for? Rails engineers? Node experts? PHP professionals?

Once upon a time, there was PHP

What is a PHP?

PHP is an older language (first incarnation in 1994), but some important sites are have been written in it (Facebook, for instance). Is it right for your project?

No.

PHP is complex, and takes a long time to learn and especially to master. While versatile, there are better options available.

Then, there was Ruby (and Rails)

So what's a Ruby?

Ruby was released in 1995, and the Rails framework was released in 2008.

Several large sites were or are written in Ruby on Rails, including Twitter, Hulu, and Shopify. So is it right for your project?

Maybe!

Ruby on Rails is still used for many projects! It offers very rapid prototpying, and a shallow learning curve.

However...

Twitter abandoned Ruby on Rails in 2008, citing scaling issues, and "rolled their own" Java framework, Blender.

If you anticipate a high amount of traffic, Rails might not be for you.

Recently, there was Node.js

Tell me about Node

Node is young, and was started in 2009. I mean, Ryan Dahl (the author) looks under 30 (but who knows, really?).

Node has had some problems, but overall has matured to be a great platform. Walmart launched a middleware written in Node right before Black Friday... and their servers were "bored out of their minds".

So is Node right for your project?

Probably!

Node is a top choice for writing modern web applications. Oh, and it also is very aware of mobile apps, too. You can just treat the website as just another app.

To Recap...

  • So, I should use PHP? NO.
  • If you want to be able to create quickly, but don't expect to scale (or are willing to port your application if you do), Rails might be for you.
  • If you want your site to be super fast, and expect to be linking up mobile apps, Node.js is probably for you.
